# @AliasFor이란?

- Spring에서 `@AliasFor` 어노테이션은 어노테이션 속성에 대한 별칭(alias)을 정의하는 데 사용되는 데이터 어노테이션이다.
- 이를 통해 어노테이션의 속성 이름을 변경하거나, 같은 속성에 대해 여러 개의 이름을 지정할 수 있다.

## AliasFor이 유용한 상황

1. **속성 이름 변경** : 기존 어노테이션의 속성 이름이 적절하지 않거나, 더 의미 있는 이름으로 변경하고 싶을 때 `@AliasFor`를 사용할 수 있다. 이를 통해 코드의 가독성과 의미를 향상시킬 수 있다.
2. **메타 어노테이션** : 어노테이션을 조합하여 새로운 어노테이션을 만들 때, `@AliasFor`를 사용하여 메타 어노테이션의 속성과 실제 어노테이션의 속성을 연결할 수 있다. 이를 통해 중복을 제거하고, 일관성을 유지할 수 있다.
3. **상속 관계** : 어노테이션 간의 상속관계에서 `@AliasFor`를 사용하여 부모 어노테이션의 속성을 자식 어노테이션에서 재정의하거나, 별칭을 지정할 수 있다.

### 사용 예시

- `@AliasFor` 어노테이션은 `annotation` 속성과 `attribute` 속성을 가지고 있다.

  - `annotation`: 별칭을 지정할 어노테이션의 클래스를 지정한다. 기본값은 `@AliasFor`가 적용된 어노테이션 자신이다.
  - `attribute`: 별칭을 지정할 속성의 이름을 지정한다. 기본값은 `@AliasFor`가 적용된 속성의 이름이다.

- 예를 들어, `@RequestMapping` 어노테이션의 `value` 속성과 `path` 속성은 `@AliasFor`를 사용하여 서로 별칭으로 지정되어 있다:

```java
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Mapping
public @interface RequestMapping {

    @AliasFor("path")
    String[] value() default {};

    @AliasFor("value")
    String[] path() default {};

    // ...
}
```

- 이렇게 함으로써 `@RequestMapping`을 사용할 때 `value` 속성과 `path` 속성을 interchangeably하게 사용할 수 있다.

- 또다른 예시로 기존에 정의되어 있는 어노테이션을 현재 상속 받은 어노테이션에서 속성을 재정의하거나, 별칭을 지정하고 싶을 때 아래와 같이 사용할 수 있다.
- 이를 통해 기존 `@Service` 어노테이션에 존재하던 value 속성을 `@Business` 라는 사용자 정의 어노테이션의 속성으로 지정해줄 수 있다.

```java
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Service
public @interface Business {
    @AliasFor(annotation = Service.class)
    String value() default "";
}
```

- `@AliasFor`는 Spring Framework 4.2 버전부터 도입되었으며, 어노테이션을 보다 유연하고 표현력 있게 만드는 데 도움을 준다.
- 특히, 메타 어노테이션을 활용한 조합 어노테이션을 만들 때 매우 유용하게 사용될 수 있다.
